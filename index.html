Meeting Notes

Last meeting, we came up with two project ideas: make a calculator or a clock. The general consensus was that both of these ideas were too basic. This meeting, we were able to come up with much more interesting ideas, such as making a basic messaging app or creating a simple mobile game.

We decided on going forward with the messaging app idea. While we were discussing what short term goals we wanted for the beginning of the project, Hank made a good point that we should try to limit our scope as much as possible, so that we would not cut out more work than we could handle.

As a team, we were able to come up with a few short term goals for the upcoming work week:
1. Set up a GitHub repository and add all members
2. Research the types of functionality that a messaging app should have at minimum
3. Use other messaging apps and see which features we would like to implement in our own design

With enough time to spare towards the end of the meeting, we were able to divide up the work between our members. Rochelle will be mainly in charge of the backend code, Hank could work on the database design, and George would take control of the frontend.
